# Contributing to dotnet-realtime-pipeline

Thank you for your interest in contributing to `dotnet-realtime-pipeline`! 

## Getting Started

1. **Fork** the repository on GitHub.
2. **Clone** your fork locally.
3. **Create a branch** for your feature or bug fix.

## Development Requirements

- **.NET 10.0 SDK** is required to build and run the project.

## Running Tests

To run the tests locally, use the .NET CLI:
```bash
dotnet test
```

## Code Style

- Please follow the existing coding conventions used throughout the codebase.
- Ensure all public APIs have appropriate **XML documentation**.
- **IMPORTANT**: Keep all existing author headers in the files. Do not remove them under any circumstances.

## Submitting a Pull Request

1. Commit your changes with clear and descriptive commit messages.
2. Push your branch to your fork.
3. Submit a Pull Request against the main repository.

## Reporting Issues

If you find a bug or have a feature request, please use **GitHub Issues**. 
When reporting an issue, please include detailed **reproduction steps** to help us resolve it quickly.
